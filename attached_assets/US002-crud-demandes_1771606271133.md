# US002 - CRUD Demandes

## User Story

**En tant que** email-parser ou frontend  
**Je veux** gerer les demandes via API REST  
**Afin de** creer et suivre les demandes d'intervention

---

## Criteres d'acceptation

### Scenario 1: Creer une demande
**Donne** un bien_id valide et des donnees completes  
**Quand** POST /api/demandes  
**Alors** la demande est creee avec etat "nouvelle"

### Scenario 2: Lister demandes d'un bien
**Donne** un bien avec 5 demandes  
**Quand** GET /api/demandes?bien_id=123  
**Alors** les 5 demandes du bien sont retournees

### Scenario 3: Filtrer par etat
**Donne** des demandes dans differents etats  
**Quand** GET /api/demandes?etat=en_cours  
**Alors** seules les demandes "en_cours" sont retournees

### Scenario 4: Bien inexistant
**Donne** un bien_id invalide  
**Quand** POST /api/demandes  
**Alors** erreur 400 "Bien non trouve"

## Endpoints

### GET /api/demandes
```
Query: page, limit, bien_id?, etat?, metier?

```

### GET /api/demandes/:id
```
Response 200: {id, bien_id, bien: {...}, ...}
Response 404: {error}
```

### POST /api/demandes
```
Body: {
    bien_id,
    objet,
    etat,
    metier,
    detail?,
    commentaire?,
    gestionnaire_id,
    date_demande_client,
    ref_syndic?,

```

## Schema base de donnees

```sql
CREATE TABLE demandes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    bien_id INTEGER NOT NULL,
    objet TEXT NOT NULL,
    etat TEXT NOT NULL DEFAULT 'nouvelle',
    metier TEXT NOT NULL,
    detail TEXT,
    commentaire TEXT,
    gestionnaire_id INTEGER NOT NULL,
    date_demande_client TIMESTAMP NOT NULL,
    ref_syndic TEXT,
    travaux_enerpur BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (bien_id) REFERENCES biens(id),
    FOREIGN KEY (gestionnaire_id) REFERENCES gestionnaires(id)
);

CREATE INDEX idx_demandes_bien_id ON demandes(bien_id);
CREATE INDEX idx_demandes_etat ON demandes(etat);
CREATE INDEX idx_demandes_created_at ON demandes(created_at);
```
